AAA Testing

Why is it important?

The AAA (Arrange-Act-Assert) pattern has practically become the industry standard. It advises that your test method be divided into three parts: organize, act, and assert. Each of them is exclusively in charge of the section for which they were named.

In this page, you will learn:

  • What is AAA testing?
  • Why AAA testing is important

The arrange step sets up the testâ€™s input values.

The act step prompts the primary function being tested.

And finally, the assert step verifies that the output of the function is what was expected.

  1. In "assert(act(arrange()));" Each step is represented by a function that is called in ascending sequence from left to right. The first command is to arrange. The arrange function returns a value, which is then provided to the act function, which returns a value, which is then sent to assert. Assert returns void, a bool, or any value you, the author, deem useful for the test.
  2. To begin, establish a separate test method that is available to all tests. One way is to construct a 'TestFixture' class that all test classes inherit.
  3. The test method can then be called inside a test. With this in place, your job becomes creating the functions to be passed as parameters.

Overall, the AAA pattern is just intended to be used as a guideline. It's a framework, or a technique of thinking about and organizing your tests such that they're easy to comprehend. There is no documentation included in any package or downloaded tool. Instead, it leaves it up to you, the author, to decide how to best produce and use it.

Developers write unit tests for their code to make sure that the code works correctly. This helps to detect and protect against bugs in the future. Sometimes developers write unit tests first, then write the code. This approach is also known as test-driven development
