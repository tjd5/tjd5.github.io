Object-Oriented Programming

Glossary

OOPs stands for Object-Oriented Programming in Python. An object is a collection of variables and functions that are linked together. These variables are frequently referred to as object attributes, and functions are referred to as object behavior. These items provide the software a more organized and logical structure.

In this page, you will learn:

  • Concepts of OOPs
  • OOPs terms

An example of an object could be a dog, the properties would be the color, breed, name, weight, age, etc. The function would include barking, walking, playing, etc.

Basic concepts of Object-Oriented Programming include:

  1. Class
  2. Object
  3. Method
  4. Inheritance
  5. Encapsulation
  6. Polymorphism
  7. Data Abstraction

A collection of objects is referred to as a class. Classes, unlike primitive data structures, are user-defined data structures. They make the code easier to work with.

Only a description or blueprint of the item is produced when a class is defined. Until we build its object, there is no memory allocation. The data or information in the objector instance is actual.

A method is a process connected with a message and an object in object-oriented programming. An object is made up of data and behavior, which are combined to provide an interface that describes how the object may be used by its many users.

The mechanism in which one class inherits the attributes and methods of another class is known as inheritance, and the child class is the class that inherits the properties from the parent class.

Glossary of OOPs terms:

  • Factory : A factory is a function or method that returns objects of a different prototype or class from a method call that is expected to be "new."
  • Instantiation : Instantiation is the process of taking a class, and creating an object from it that you can actually use in your program.
  • Object : A class instance is referred to as an object. In OOPS, an object is a self-contained component that has methods and attributes that make a certain sort of data usable.
  • Class : in object oriented programming, a class is a blueprint or prototype that defines the variables and the methods.
  • Namespace : A namespace is a notion in object-oriented programming that allows variables, functions, and other program components to be organized. A namespace isolates code such that you must first refer to the namespace you're referring to in order to access it.
  • Constructor : A constructor is a special method of a class or structure in object-oriented programming that initializes a newly created object of that type. Whenever an object is created, the constructor is called automatically.
  • Fixtures : Fixtures let you pre-populate your testing database with data before running your tests.
  • Type Hint : Type hinting is a formal technique for indicating the type of a value in Python code statically. The name: str syntax denotes that the name parameter must be of the str type. The greet() method will return a string, as indicated by the "->>"Â syntax.
  • Static : A static method belongs to a class, but it does not belong to an instance of that class, and it can be called without the class's instance or object.
  • Class Method : In object-oriented programming, a method is a technique that is connected with a class. The behavior of objects produced from a class is defined by a method. Another way of putting it is that a method is a set of actions that an object may execute.
  • Facade : A typical software design pattern in object-oriented programming. A facade is a small piece of code that offers a streamlined interface to a larger body of code, such as a class library.
  • Design Patters : A design pattern is a generic, reusable solution to a problem that arises frequently in software development. The pattern usually depicts the interactions and links between classes or items.
  • Property : In various object-oriented programming languages, a property is a type of class member that sits halfway between a field (or data member) and a method in terms of capability.
