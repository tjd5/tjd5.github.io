Calculator Demonstrations

How are encapsulation, inheritance, polymorphism, and abstraction seen in the calculator program?

In this page, you will learn:

  • What is demonstrated in the calculator program?
  1. Encapsulation

    Encapsulation is seen in the calculator program. We use calculators everyday yet we do not know how it works on the inside. When we type in a command such as 1+1, we get an answer on the display instantly.

    Encapsulation states that all we need to know about a class is its interface in order to utilize it. To utilize it, we don't need to know anything about the class's internals.

  2. Inheritance

    Inheritance allows for programs to have classes that are built upon other classes.

    Inheritance in the calculator program the classes to each operator is inherited inside the calculator class.

  3. Polymorphism

    In the calculator, a parent class (calculator class) is used to refer to a child class (operator classes).

    Using polymorphism, although the amount of code has increased, the logic is more clear, and you only need to focus on the operation itself.

  4. Abstraction

    Through the process of abstraction, a programmer hides all but the relevant data about an object in order to reduce complexity and increase efficiency.

    Using abstraction in the calculator program, to activate a given operation, you only need to know which methods of the object are accessible to call and which input arguments are required. However, you don't need to know how this method is built or what steps it must take to get the desired outcome.
