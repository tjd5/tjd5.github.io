Using Docker

Using GitHub, Terminal, and Azure

In this section of the website, you will learn about Docker. Docker is a containization tool that will upload your website to Azure. We will also go over Docker Commands.

In this page, you will learn how to:

  • Build and run an image as a container
  • Share images using Docker Hub
  • Deploy Docker applications using multiple containers with a database
  • Running applications using Docker Compose

The first step to getting started with Docker is to download and install the current version of Docker on your computer. Before doing so, make sure that you are installing the correct Docker for your computer (Apple chip/Intel chip/Windows/Linux).

Once Docker is installed on your computer, open command prompt or terminal depending on what operating system you have. Next, we will start imputing commands to run docker.

First run the command: $ docker run –d –p 80:80 docker/getting-started.

  • The –d runs the container in the background which is called detached mode.
  • The –p 80:80 maps the port 80 of the host to port 80 of the container.
  • Lastly, docker/getting-started is the image that is being used.

After this, you can open your Docker Desktop and see the container that is running on your machine. The Docker Desktop is helpful to see container logs, getting shells inside the container, and efficiently manage containers. A container is another process that is running on your machine separate from the processes on host.

Before running the application, the application source code must be on your machine. Clone the repository that contains your application source code and use any code editor that you would like to open the project. To build the application, it is important to understand that a Dockerfile is needed. A Dockerfile is a script of instructions that is used to create the container image.

Dockerfiles are typically set up using four layers that are stacked with the layers being:

  • FROM (environment): creates a layer from the environment's Docker image.
  • COPY (./app): adds the files from the current directory from the Docker client.
  • RUN (make /app): builds your application.
  • CMD ( ): specifies what command to run in the container.

All the text in the parenthesis can be different based on what you are using, it is important to read up on the best practices for writing Dockerfiles to understand how to correctly execute a working Dockerfile. (NOTE: MAKE SURE THE DOCKERFILE DOES NOT HAVE A .TXT EXTENSION AS THIS WILL CAUSE ISSUES LATER)

Using your terminal, and make sure that you are in the app directory with the Dockerfile. Now, you can build the container using the command: $ docker build -t getting-started . This command builds a new container image using the Dockerfile.

Next, you can start your container using the docker run command and specify the name of the image created, for example: $ docker run -dp 3000:3000 getting-started

Using this example, you can now open up a web browser and type in http://localhost:3000. This should show the application.

You can use the command: $ docker ps, to get the ID of the container. Using this ID you will now be able to stop the container by using the command docker stop. The command is: $ docker stop

When you are ready, you can also share this application on Docker Hub. To push the image, you need to create a repository on Docker Hub making sure that the visibility is public. Next you need to use the command: $ docker login -u YOUR-USER-NAME to login to Docker Hub. Next, use the docker tag command: $ docker tag getting-started YOUR-USER-NAME/YOUR-REPO-NAME. Lastly, you can push your repository to Docker Hub using the command: $ docker push YOUR-USER-NAME/YOUR-REPO-NAME.
